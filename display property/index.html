Display property

What is the display?

This is one of the property, briefly saying, it is to decide the element display format
There are block, inline, inline-block and none.

What is Block?

Block elements break the flow of a document. Width, height, margin and padding are respected p,div,ul, and h1 to h6 are default settings by block.

What is the inline

Width and height are ignored. Margin and padding puch elements away HORIZONTALLY but not VERTICLLY a,span and image tag are default settings by inline.



1. Mainly used as part of the text

For example, it is used for elements such as text decorations and links.



2. Lined up side by side with no line breaks between elements

As with the text, when you reach the right end, a line break will occur.

This is a tag and span tag.

In this way, before and after the inline element, there are no line breaks and they are lined up side by side.



3. Basically used in block element

On the contrary, it is rare to put a block in the inline.



4. Cannot specify width and height

You cannot specify the width and height of the inline element with CSS. The width and height of the inline element depends on the "contents of the element". In other words, the width and height are automatically determined by the "text length" and "letter size".

5. Basically used in block element

On the contrary, it is rare to put a block in the inline.



6. Can't adjust the top and bottom margins

In inline element, the left and right margins which are padding and margin can be freely specified.

However, you cannot specify the top and bottom margins. Also, although you can specify the top and bottom padding, if you increase the padding, it will overlap with the previous and next lines when a line break occurs, and the design will collapse.

So the most important here is you can specify only the right and left margin as the inline element.

The horizontal padding and margin are properly specified. On the other hand, the top and bottom margins are invalid. Although the top and bottom padding is reflected, it overlaps with the previous line, resulting in a chaotic situation.

What is the inline-block

Behave like an inline element except width, height, margin and padding are respected This is something in between inline and block. Roughly speaking, the order of the elements is inline, and the contents of the elements are block-like in nature.



1. Width and height can be specified

The width and height can be freely specified in the same way as block.



2. You can also specify the top, bottom, left, and right margins

The margins are the same as block, and you can freely specify the margin and padding values.



3. Can also specify text-align and vertical-align

With inline-block, you can also specify "text-align" and "vertical-align". It's kind of a nice combination, isn't it?
